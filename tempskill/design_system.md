# Finance Report Design System

## Purpose
This document defines the visual and editorial style for all finance reports generated by the skill.

The goal is to produce reports that feel refined, calm, modern, and highly readable, inspired by a minimalist premium-product aesthetic.

---

## Design Principles

1. **Clarity first**  
   Every section must be immediately understandable.

2. **Minimalism over decoration**  
   Avoid unnecessary visual noise, emojis, excessive icons, or overly dense layouts.

3. **Strong hierarchy**  
   Titles, subtitles, metrics, and commentary must feel clearly separated.

4. **Whitespace matters**  
   Use generous spacing between sections and blocks.

5. **Calm and premium tone**  
   The report should feel polished, thoughtful, and executive-ready.

---

## Typography Guidance

Use a clean, modern sans-serif tone in the writing style.

Typography hierarchy:

- **Title**: short, strong, premium
- **Section headings**: concise and descriptive
- **Body text**: simple, precise, readable
- **Metric labels**: subtle and clear
- **Key figures**: visually emphasized

Writing style should feel:
- concise
- elegant
- informative
- professional

Avoid:
- overly technical jargon
- playful language
- excessive bold formatting
- all caps except for small labels when necessary

---

## Color Intent

Even if the output is plain markdown or structured text, the style should assume the following color logic for future rendering:

- **Primary text**: near-black
- **Secondary text**: soft gray
- **Background**: white or very light neutral
- **Positive values**: restrained green
- **Negative values**: restrained red
- **Neutral accents**: cool gray or muted blue

Color usage should remain subtle. Never make the report feel loud or overly “dashboard-like.”

---

## Layout Rules

Each report should follow this structure:

1. **Title**
2. **Report metadata**
3. **Executive summary**
4. **Key metrics snapshot**
5. **Income analysis**
6. **Expense analysis**
7. **Monthly trend review**
8. **Top spending categories**
9. **Notable transactions**
10. **Key takeaways / recommendations**

---

## Section Formatting Rules

### Title
- Keep it short and premium
- Example: `Quarterly Finance Overview`
- Add a subtitle with period and account holder if available

### Report metadata
Include:
- account holder
- reporting period
- currency
- opening balance
- closing balance

### Executive summary
- 1 short paragraph
- Summarize the overall financial picture
- Mention whether the period is healthy, stable, improving, or concerning

### Key metrics snapshot
Present the most important figures clearly:
- total income
- total expenses
- net change
- savings rate

Metrics should be easy to scan and separated from narrative text.

### Income analysis
- Summarize main income sources
- Mention recurring vs non-recurring income
- Call out unusual income events

### Expense analysis
- Summarize spending behavior
- Highlight major categories
- Mention large one-time expenses
- Distinguish essential vs discretionary spending when possible

### Monthly trend review
- Review month-by-month performance
- Identify positive or negative trends
- Explicitly mention any month with negative net balance

### Top spending categories
- Rank categories from highest to lowest
- Add brief interpretation, not just numbers

### Notable transactions
Highlight important transactions such as:
- large expenses
- unusual purchases
- high-value income events

### Key takeaways / recommendations
- End with 3 to 5 concise insights
- These can include patterns, alerts, strengths, or improvement opportunities

---

## Tone Rules

The report should sound like an experienced financial analyst writing for a professional reader.

Tone should be:
- calm
- clear
- objective
- polished
- slightly editorial, but never dramatic

Avoid:
- alarmist phrasing
- overly casual expressions
- exaggerated claims

Preferred phrasing examples:
- “Spending remained concentrated in housing and discretionary purchases.”
- “March shows a temporary negative balance driven by travel and electronics expenses.”
- “Income remained stable, with additional support from freelance work.”

---

## Formatting in Markdown

Use markdown with clean hierarchy:

- One `#` title only
- `##` for main sections
- `###` only when needed
- Short paragraphs
- Bullet lists for highlights
- Tables only if they improve readability
- Bold only for metric labels or key values
- Avoid heavy nesting

---

## Consistency Rules

Every generated report must:
- follow the same section order
- use the same tone
- present metrics in a consistent way
- maintain visual simplicity
- prioritize readability over exhaustiveness

If some data is missing, keep the structure and explicitly mention assumptions or unavailable values.

---

## Output Quality Standard

A good report should feel:
- ready to share internally
- visually coherent
- analytically useful
- consistent across different JSON inputs

The result should look like a premium, structured briefing rather than a raw AI response.
